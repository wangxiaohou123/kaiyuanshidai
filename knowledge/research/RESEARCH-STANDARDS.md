# 开元时代 · 研究方法论标准

> **制定日期**：2026-03-14
> **制定背景**：CEO王晶晶明确要求——所有研究必须达到「一手数据调研」颗粒度，以大众点评实时爬取为基准，不接受仅引用二手分析报告的研究。
> **适用范围**：所有研究院T系列课题、日报情报补充研究、战略决策支撑研究

---

## 核心原则

> **"这些第一手的调研信息，才是我们做研究的方式。请以后未来的研究要到这个颗粒度。"**
> — CEO 王晶晶，2026-03-14

### 原则一：一手数据优先

每项研究必须有至少一类一手数据来源，优先级：
1. **点评/实时平台爬取**（大众点评、美团、口碑）
2. **广告账户数据**（美团广告账户报表）
3. **招聘数据**（BOSS直聘、拉勾、猎聘）
4. **现场调研**（实地走访、电话咨询）
5. **政府公开数据**（民政局公示、医保局文件）

### 原则二：数据颗粒度要求

#### 竞争格局研究（必须包含）

**基础数据**（每家机构必采）：
```
□ 门店地址（到街道级，含地铁站步行距离）
□ 点评评价条数（评价量反映服务活跃度）
□ 点评人均消费（注意：不等于真实客单价，需解读）
□ 点评评分（如显示）
□ 团购产品：名称 + 价格 + 年售量（0的特别标注）
□ 美团广告CPC投放量（来自账户报表）
```

**深度数据**（重点竞对，至少爬取TOP5竞对）：
```
□ 用户评价标签前5项（量化用户关注点）
□ 代表性5星评价原文（至少2条）
□ 代表性差评原文（至少1条，找竞对弱点）
□ 评价时间分布（判断近3个月是否活跃）
□ 门店截图（存档备用）
```

**业务情报**（补充搜索）：
```
□ 护工/员工招聘薪资（判断人力本结构）
□ 融资轮次和估值（判断资本实力）
□ 工商注册状态（成立年份/经营状态）
□ 城市扩张速度（过去12个月新开店数）
```

### 原则三：数据交叉验证

单一数据源得出的结论必须用另一数据源验证：
- CPC投放量 × 点评评价数 → 验证业务真实活跃度
- 点评人均 × 招聘薪资 → 还原真实服务收费结构
- 团购年售量 × 门店评价数 → 判断线上/线下流量来源

**典型案例**：南山锦阁点评人均¥30，CPC=0，团购年售0 → 三个数据共同指向「餐饮门店而非养老服务门店」的结论。仅靠单一数据会误判。

### 原则四：结论可追溯

每个关键结论必须注明数据来源：
- ✅ "福寿康虎坊桥店年售0次（点评实测，2026-03-14）"
- ❌ "福寿康在广外附近影响力有限"（无数据支撑的主观判断）

## 研究流程 SOP

### 阶段一：准备（30分钟）

1. 明确研究问题（什么决策需要这个研究？）
2. 列出目标机构名单（从广告账户CSV中提取）
3. 规划数据采集路径（点评搜索词设计）

### 阶段二：一手数据采集（1-2小时）

**大众点评批量爬取**：
```bash
# 1. 确认浏览器登录状态
agent-browser get url

# 2. 搜索目标机构
agent-browser open "https://www.dianping.com/search/keyword/2/84_{机构名}"

# 3. 从搜索页获取批量数据（评价数+人均+评分）
agent-browser snapshot | python3 -c "解析脚本"

# 4. 进入重点机构详情页
agent-browser open "https://www.dianping.com/shop/{shop_id}"
agent-browser screenshot /tmp/dp_{机构简称}.png

# 5. 爬取团购产品信息
agent-browser scroll down 300
agent-browser screenshot /tmp/dp_{机构简称}_prices.png
```

**评价文本提取**：
```bash
agent-browser snapshot | grep -E "评价|星|标签|服务|护理|陪伴"
```

### 阶段三：数据整理与交叉验证（30分钟）

用Python整合所有数据，建立统一格式的竞对数据表：
```python
fields = [
    "机构名称", "所在城市", "所在商圈/行政区",
    "机构类型",  # 居家/机构/驿站/平台
    "点评评价数", "点评人均消费", "点评评分",
    "代表性团购产品", "团购年售量",
    "美团CPC投放量",
    "用户评价TOP标签",
    "代表性用户评价原文",
    "研究员注释"
]
```

### 阶段四：研究报告输出

**文件命名规范**：`T{编号}-{研究主题}-{版本}.md`
- 例：`T10-competition-dianping-fieldwork-v1.md`

**报告必须包含章节**：
1. 研究背景与目标
2. 各机构详细数据（按上述颗粒度）
3. 数据交叉分析
4. 关键结论（可追溯到原始数据）
5. 研究方法记录（数据来源+采集方式）
6. 待补充项（下一步需要的数据）

## 质量控制清单

研究提交前的自查清单：

```
□ 是否有至少一类一手数据？
□ 每个关键结论是否可追溯到数据来源？
□ 是否对竞对TOP5进行了点评深度爬取？
□ 是否记录了截图文件路径？
□ 是否有数据交叉验证（至少2个数据源互证关键结论）？
□ 是否列出了"待补充项"（承认研究局限性）？
□ 结论是否区分了"事实"和"推断"？
```
